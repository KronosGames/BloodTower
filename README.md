## プログラムでやってはいけないことリスト
* var型は、使用禁止
   * 理由：var型は、膨大なメモリを取り、どれでも当てはまるようなメモリを使っています。
* 関数からの変数アクセス＆配列アクセスは、禁止
   * 例えば、GetWeaponList()[0] や　GetWeapon().id , GetComponent<Character>().attack など
   * 理由：関数を呼ぶと呼び出し負荷がかかります。上記みたいなことやると、2度関数にアクセスすることになります。
* 3つ以上の分岐でif文の使用は、禁止
   * if文は、2分岐探索で処理します。なので、2つのみの分岐なら、if分で大丈夫です。
   * でも、3つ以上になるとswitch文の方が高速に処理します。
* foreach 禁止
   * 理由：配列を１つのデータとして扱うために、IEnumrableが作り出します。
   * その時点で、GCが発生し、パフォーマンスが落ちてしまいます。
* まぁ、色々ありますが、その都度言っていきます。
